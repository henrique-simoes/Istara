# Sage -- UX Evaluation Agent Memory

## Learnings Log

### Key UX Findings
- The onboarding flow assumes users understand the Atomic Research model. Most junior researchers don't. Need a brief explainer
- Transition from Chat to Findings view loses context. Users expect to see the findings that were just generated, not the full findings list
- Empty states across the platform are informational but don't provide clear CTAs for next steps
- The agent selection in chat doesn't clearly communicate how different agents will respond differently

### User Persona Notes
- **Junior Researcher**: Needs guidance at every step. Values templates and examples. Overwhelmed by the number of skills available
- **Senior Researcher**: Values efficiency and flexibility. Wants keyboard shortcuts, batch operations, and custom workflows
- **Research Manager**: Focuses on oversight and reporting. Needs dashboard views, project health indicators, and exportable reports
- **Stakeholder**: Needs simplified views of research outputs. Shouldn't need to understand the research process to consume insights

### Platform Strengths
- The skill catalog is comprehensive (35+ UXR methods)
- The context hierarchy system is architecturally sound
- A2A messaging infrastructure enables future collaborative agent workflows
- The Atomic Research model provides strong evidence traceability

### Improvement Priorities
- First-time user experience needs guided onboarding
- Cross-feature navigation needs clearer wayfinding
- Agent personalities should be distinct and visible to users
- Research workflow templates would reduce setup friction for common research patterns
